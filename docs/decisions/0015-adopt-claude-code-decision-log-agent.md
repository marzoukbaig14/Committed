## Context
ADR 0014 left Claude Code available (via the Northeastern Enterprise premium seat, at no
personal cost) but deferred by choice during the learning phase. The terminal CLI has since been
confirmed working in the Codespace, authenticated against the Enterprise seat. Logging ADRs by
hand (create the record, run the generator, commit) is repetitive bookkeeping that has now been
done manually enough times to be understood. The project already defines a decision-log agent
role with a brief (handoffs/DECISIONLOG_AGENT.md) that had no implementation.

## Decision
Adopt Claude Code (terminal CLI, in the Codespace) as the implementation of the decision-log
agent, scoped to the decision domain only: creating ADR record files and regenerating
docs/DECISION_LOG.md and docs/decision-tree.md. Claude Code performs the mechanics; the decision
and its rationale continue to be worked out by the human in chat. Claude Code is not used for
core work (data filter, training config, eval metrics, judge prompt), which stays hand-written
for learning. This is the last ADR created by hand; subsequent ADRs are created by Claude Code
following handoffs/DECISIONLOG_AGENT.md.

## Consequences
Future ADRs are written and the log regenerated by Claude Code rather than by hand, removing the
mechanical bookkeeping while keeping the reasoning with the human. No new cost (covered by the
Enterprise seat). Claude Code does not sync the chat's Project knowledge, so STATUS.md plus a
manual paste remains the continuity mechanism (per 0014). Amends the Claude-Code-deferred stance
of 0014 (the rest of 0014 stands) and fills the role briefed in handoffs/DECISIONLOG_AGENT.md.
Relates to ADR 0001.
